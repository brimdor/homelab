API Reference

REST API endpoint documentation for the PageVault platform

Authentication

All API requests require a Bearer token in the Authorization header. Tokens are issued via SSO Provider OAuth flow and scoped to the user's vault.

Authorization: Bearer eyJhbGciOiJSUzI1NiIs...
GET /api/v1/vaults

List all vaults

Returns all vaults accessible to the authenticated user. Supports pagination via cursor-based tokens.

limitinteger — Max vaults to return (default 25, max 100) cursorstring — Pagination cursor from previous response

Response

{ "vaults": [ {"id": "vault_abc123", "name": "Engineering", "repo_url": "...", "created": "2026-03-15T10:30:00Z"} ], "next_cursor": "eyJsaWIiOi" }
POST /api/v1/vaults

Create a new vault

Provisions a new vault with an associated Git repository. The repository is initialized with a README and default branch configuration.

namestring (required) — Display name for the vault descriptionstring — Optional vault description ai_providerenum — "openai" | "ollama" | "anthropic" (default: "ollama")

Response

{ "id": "vault_def456", "name": "Research Notes", "repo_url": "https://git.acme-platform.io/repos/vault-abc123.git", "status": "provisioning" }
GET /api/v1/vaults/:id/chat

Chat with vault

Send a message to the AI chat engine scoped to a specific vault. Responses are streamed via SSE for real-time token delivery.

messagestring (required) — User message content modelstring — Override default model for this request streamboolean — Enable SSE streaming (default: true)
DELETE /api/v1/vaults/:id

Archive a vault

Marks a vault as archived. The Git repository becomes read-only. Chat and sync are disabled. Archival is reversible within 30 days.
