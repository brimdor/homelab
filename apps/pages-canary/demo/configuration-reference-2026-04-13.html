CANARY

Configuration Reference

Environment variables and Helm values for PageVault deployments

Core Environment Variables
GITHUB_OWNER GitHub organization or user that owns vault repositories Required
GITHUB_REPO Default repository name for new vaults Required
GITHUB_TOKEN Personal access token or GitHub App installation token with repo scope Required
OLLAMA_BASE_URL Base URL for local Ollama inference server http://ollama:11434
OPENAI_API_KEY OpenAI API key for cloud AI fallback Optional
VAULT_DIR Root directory for vault Git clones on the filesystem /tmp/vaults
PORT HTTP listen port for the Express server 3000
Helm Values (values.yaml)
image.repository Container image registry path registry.internal:5000/pagevault
image.tag Image tag to deploy (date-stamped for canary) latest
ingress.hosts[0].host Public hostname for the application pagevault.acme-platform.io
persistence.data.type Storage type — nfs (prod) or omitted (canary/ephemeral) nfs
persistence.data.server NFS server IP (only when type=nfs) nfs.internal
persistence.data.path NFS export path on the server /data/pagevault
resources.limits.memory Maximum memory allocation 2Gi
resources.requests.cpu Minimum CPU guarantee 100m
GitOps Sync Options
syncPolicy.automated Enable auto-sync and self-heal true / true
syncOptions CreateNamespace=true ensures namespace exists before deployment CreateNamespace=true
