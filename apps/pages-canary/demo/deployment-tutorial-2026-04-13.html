Deployment Tutorial

Ship your app to the Kubernetes cluster with GitOps

1

Create the Helm Chart

Each application lives in apps/<app-name>/ in the infrastructure repository. Start with the app-template as a base.

apps/my-app/ ├── Chart.yaml # app-template v4.6.2 dependency ├── values.yaml # deployment config (image, ingress, PVC) └── templates/ # optional custom manifests
2

Configure values.yaml

Define your container image, ingress hostname, and any persistent volumes. The app-template handles the boilerplate.

app-template: controllers: main: containers: main: image: repository: registry.internal:5000/my-app tag: "latest" ingress: main: hosts: - host: my-app.acme-platform.io
Tip: Always add cdn-proxied: "true" to ingress annotations so external-dns creates proxied DNS records through the CDN.
3

Build and Push the Image

Container images are built with Docker and pushed to the private Container Registry on the platform network.

docker build -t registry.internal:5000/my-app:latest \ -f Dockerfile . docker push registry.internal:5000/my-app:latest \ --tls-verify=false --quiet
Warning: Never use kubectl apply directly. The GitOps controller auto-heals and will revert manual changes. Push to Git Server instead.
4

Commit to Git Server

Push your Helm chart changes to the infrastructure repo on Git Server. The GitOps controller watches this repo and will automatically sync within 1-2 minutes.

git add apps/my-app/ git commit -m "feat: add my-app deployment" git push origin main
5

Verify the Deployment

Check that the GitOps controller picked up the change and the pod is running healthy.

kubectl get pods -n my-app kubectl get ingress -n my-app curl -sI https://my-app.acme-platform.io/ | head -3
Tip: If external-dns isn't creating the DNS record, check for stale CNAME conflicts.
