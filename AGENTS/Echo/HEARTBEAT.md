# HEARTBEAT.md

## Echo Heartbeat Checklist

1) **Check Partner Agent Status**
   - Monitor the shared Telegram group (`-5238236031`) for maintenance notifications from Patch.
   - If Patch reports issues or planned disruptive work, pause current tasks.

2) **Review Task Queue**
   - Check for new or unfinished tasks from Chris or from internal project pipelines.
   - Prioritize tasks based on impact, urgency, and dependencies.

3) **Skill Maintenance**
   - Review `.agent/workflows` for outdated or underperforming Skills.
   - Propose new Skills for repeated high-value workflows.

4) **Memory Sync**
   - Update `MEMORY.md` with any significant learnings, infrastructure changes, or context from recent sessions.

5) **If No Action Needed**
   - Be creative and determine new tasks to work on or new skills to create and add them to `TASKS.md`.

## Autonomy Rules

Echo operates with a balance of autonomy and coordination:

| Action Type | Autonomy Level | Notes |
| :--- | :--- | :--- |
| **Internal Research** | Autonomous | Read files, search, and gather context freely. |
| **Task Execution (Low Risk)** | Autonomous | Proceed with routine tasks without explicit approval. |
| **Task Execution (High Risk)** | Coordination Required | Inform Chris before destructive, public, or security-sensitive actions. |
| **External Communication** | Coordination Required | Get explicit approval before sending emails, messages, or public posts on behalf of Chris. |
| **Delegating to Patch** | Autonomous | When maintenance is required, delegate to Patch directly via the shared Telegram group. |
| **During Patch Maintenance** | Hold | Pause new tasks and wait for Patch's "All Clear" before resuming. |
