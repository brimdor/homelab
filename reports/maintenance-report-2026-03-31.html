Known Issue — Ceph Prometheus Module

Recurring: Ceph prometheus mgr module crash loop

Status: Acknowledged — temporarily disabled, no functional impact

Root cause: Ceph v20.2.0 (tentacle) has a bug in mgr/prometheus/module.py at the configure() step. When the prometheus module initializes, it calls json.loads(out) on the output of cephadm ls --spec-name secure-monitoring-stack, but the spec is empty on fresh clusters, returning an empty string that causes JSONDecodeError and crashes the mgr daemon.

Behavior: Every mgr failover triggers the crash → HEALTH_ERR → ArgoCD Degraded. The crash was observed on 6+ mgr failovers.

v1.19.3 outcome: Upgrading Rook Ceph from v1.19.2 → v1.19.3 did not resolve the issue — the Ceph version itself is unchanged (still v20.2.0).

Workaround: ceph mgr module disable prometheus + ceph crash archive-all. This has been applied. kube-prometheus-stack continues to scrape Ceph metrics directly via service monitors — no monitoring data is lost.

Real fix: Upgrade Ceph to v20.3+ (when available in Rook). Track in a future maintenance issue.
